## Overview

Given symptoms → identify likely layer/protocol cause → propose step-by-step checks.\
\

---

## 1️⃣ Completion Definition

Student applies TCP/IP reasoning, IP logic, protocol mapping, and DNS understanding to diagnose multi-factor network scenarios.

## 2️⃣ Minimum Evidence

Given 2 integrated diagnostic cases, student identifies:\
Probable failure layer\
Addressing issue (if present)\
Protocol/service mismatch (if present)\
A justified corrective action

## 3️⃣ Bloom's Taxonomy Level

Analyze → Evaluate

## 4️⃣ Readiness Gate

Can the student construct a coherent diagnostic chain (symptom → layer → cause → fix) rather than listing disconnected facts? Yes/No
