# L121 — TCP/IP model as a reasoning tool (not memorization)

**Type:** 📖 Lesson
**Grade:** Grade 11
**Module:** Data – Presentation, AI & Ethics
**Semester:** Semester 2
**Sequence:** 121 of grade year
**Status:** 🚧 WIP

---

## Overview

What each layer “does” in plain operational terms; where typical problems live.\
\

---

## 1️⃣ Completion Definition

Student can use the TCP/IP layered model as a diagnostic reasoning framework (application, transport, internet, network access) to localize where a failure likely occurs, rather than reciting layer names.

## 2️⃣ Minimum Evidence

Given 5 network problem scenarios (e.g., DNS failure, cable unplugged, wrong IP config, blocked port, server down), student assigns the most probable layer and justifies reasoning using functional layer responsibilities.

## 3️⃣ Bloom's Taxonomy Level

Understand → Analyze

## 4️⃣ Readiness Gate

Can the student explain *why* a problem belongs to a given layer based on responsibility (e.g., “name resolution = application layer”) rather than guessing? Yes/No
