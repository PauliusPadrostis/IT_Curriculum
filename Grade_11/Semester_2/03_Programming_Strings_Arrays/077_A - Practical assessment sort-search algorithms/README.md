## Overview

## 1️⃣ Completion Definition (Observable Capability)

Student independently completes an assessment set that demonstrates:

- Correct reasoning about arrays/strings/sort/search behavior

- Ability to trace execution and predict outputs/state

- Ability to identify algorithm preconditions (sorted vs unsorted)

- Ability to diagnose a flaw category (bounds, swap logic, accumulator init, condition placement)

- Ability to write **one short code fragment** (10–25 lines) that implements a specified sub-task correctly

The short code component must be aligned with the reasoning portion (same domain).

## 1️⃣ Completion Definition

Student can analyse practical assessment: sort/search algorithms.

## 2️⃣ Minimum Evidence

Student submits:

**Part A — Reasoning Set (short-answer format):**

- ≥2 tracing/output prediction items

- ≥1 “identify flaw” item (label error type + 1-sentence justification)

- ≥1 algorithm-choice justification item (linear vs binary; sort-first logic)

**Part B — Short Code Task:**

- Implement one of:

  - linear search with not-found handling

  - one pass/variant of a simple sort

  - sort + search wrapper with correct preconditions

- Must run correctly on provided sample input

- Output matches spec exactly

No scaffolds.\
No partially filled loop bodies.
