## Context

- Split program into:

  - input()

  - compute()

  - output()

- Reinforce parameter passing and return values

## Completion Definition

Student decomposes a multi-step problem into logically separated functions, assigns clear responsibilities to each function, and integrates them into a coherent program architecture.

## Minimum Evidence

Student:\
identifies at least two logically distinct subtasks\
implements each subtask as a separate function\
passes parameters correctly between functions\
avoids code duplication\
ensures each function performs a single clear responsibility\
produces correct final output after integrating components.

## Cognitive Level (Bloom's)

Analyze → Apply (structural decomposition and modular design).

## Readiness Gate

Program functions operate independently and together produce correct final results; student can explain why decomposition improves clarity or maintainability.
