**Semester:** Semester 1
**Sequence:** 11 of grade year
**Lessons allocated:** 3
**Status:** 🚧 WIP

## Context

**Goal:** Remove fear + build execution confidence.\
\

## Sub-topics

- What a program is (input → process → output model)

- Development environment basics (run, error messages, print)

- Variables + assignment

- Basic data types (int, float, string — concept level)

- First debugging habit: predict → run → compare

- Input / output

## Completion Definition

Student understands fundamental program structure (input → processing → output), variables, basic data types, and sequential execution, and can write simple linear programs without relying on copy-pattern imitation.

## Minimum Evidence

Student:\
writes small programs with clear input, processing, and output sections\
correctly declares and uses variables\
performs basic arithmetic operations\
predicts output of short linear code snippets\
avoids syntax errors in simple sequential programs.

## Cognitive Level (Bloom's)

Understand → Apply (procedural logic foundation).

## Readiness Gate

Given a short linear program, student can correctly trace and predict final output without executing it.
