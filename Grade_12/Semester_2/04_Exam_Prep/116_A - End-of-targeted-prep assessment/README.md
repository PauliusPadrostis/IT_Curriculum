## Overview

### 1️⃣ Completion Definition (Observable Capability)

Student independently completes a graded assessment that:

- Integrates Data and/or Programming tasks previously trained in T1–T12

- Demonstrates stable execution under exam-realistic time pressure

- Shows correct method selection (Data) and correct algorithm composition (Programming) where required

- Produces fully specification-compliant outputs

This assessment validates readiness to move from **targeted conditioning** to **mock exam phase**.

---

## 1️⃣ Completion Definition

Student can analyse end-of-targeted-prep assessment.

## 2️⃣ Minimum Evidence

Student submits under graded conditions:

- Completed Data artifact (if included)

- Completed Programming solution (if included)

- Exact output formatting in both modalities

- Completion within defined time threshold

- No scaffolding or mid-assessment correction

Assessment must include:

- At least one non-trivial decision point (method/tool selection)

- At least one precision-sensitive formatting requirement

---

## 3️⃣ Bloom's Taxonomy Level

**Bloom: Apply → Analyze**

- Apply full-stack execution skills

- Analyze task requirements to select correct workflow

This is performance validation, not new learning.

---

## 4️⃣ Readiness Gate

✔ All must be true:

- All required sections attempted

- No runtime errors (Programming)

- No structural errors (Data grouping/pivot misuse)

- Output formatting exact

- Completed within defined time threshold

Failure indicates targeted-prep instability and requires remediation before mock phase.

---

## 5️⃣ Common Failure Modes

- Regression to earlier parsing or formatting errors

- Time misallocation between sections

- Choosing incorrect aggregation method (Data)

- Minor formatting drift under pressure

- Logical correctness but output mismatch

- Not verifying before submission
