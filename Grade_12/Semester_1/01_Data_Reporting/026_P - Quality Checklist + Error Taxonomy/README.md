## Overview

### 1️⃣ Completion Definition (Observable Capability)

Student independently audits a completed data practical task and:

- Identifies structural, logical, and reporting errors using a formal checklist

- Classifies each error into a defined error category (e.g., structural, aggregation, selection logic, visualization, interpretation)

- Proposes a precise corrective action for each detected error

Student demonstrates diagnostic competence, not re-execution.

---

---

## 1️⃣ Completion Definition

Student can produce quality checklist + error taxonomy.

## 2️⃣ Minimum Evidence

Student submits:

- Annotated version of a completed data task (own or provided flawed sample)

- Error log table containing:

  - Error description

  - Error category

  - Location (sheet/section)

  - Corrective action

- At least 4 distinct error types identified

No teacher-guided pointing to error locations.

---

## 3️⃣ Bloom's Taxonomy Level

**Bloom: Analyze → Evaluate**

- Analyze output for structural/logical correctness

- Evaluate severity and impact of errors

- Propose justified corrections

Within Grade 12 exam-conditioning scope (error recognition competence).

---

## 4️⃣ Readiness Gate

✔ All must be true:

- ≥4 distinct error categories correctly identified

- Each error mapped to correct taxonomy category

- Proposed correction is technically valid

- No misclassification of error type

If errors are vague (“wrong chart”) or unclassified → Gate not met.

---

## 5️⃣ Common Failure Modes

- Describing symptoms, not structural cause

- Misclassifying aggregation errors as formatting issues

- Ignoring refresh dependency errors

- Failing to distinguish data error vs interpretation error

- Proposing correction that does not solve root issue
