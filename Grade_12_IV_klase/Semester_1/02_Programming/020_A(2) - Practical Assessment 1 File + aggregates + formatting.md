## Completion Definition

Student independently completes a graded practical task that requires:

- Correct interpretation of programming specification

- File input and robust parsing

- Accurate computation of required aggregates

- Exact output formatting (ordering, spacing, precision)

- Safe handling of edge cases specified in task

This assessment validates foundational programming competence: file realism + correctness + format discipline.

## Minimum Evidence

Student submits under graded conditions:

- Complete working program

- Input file (provided in assessment)

- Produced output (exact format)

- No scaffolding or correction during execution

Output must:

- Match expected aggregates exactly

- Match formatting exactly (no extra spaces/lines)

- Run without runtime errors on valid input

## Readiness Gate

✔ All must be true:

- Program compiles/runs without runtime error

- Correct record count parsed

- Aggregates mathematically correct

- Output format exactly matches specification

- Safe behavior on defined edge case (if included)

Failure in formatting or aggregate correctness results in grade penalty according to rubric.

## Common Failure Modes

- Minor formatting mismatches (extra spaces/newlines)

- Incorrect initialization of aggregates (min/max logic errors)

- Division precision mistakes

- Failing to trim input whitespace

- Not handling empty dataset safely

- Printing debug output accidentally

- Hardcoding assumptions about record count
