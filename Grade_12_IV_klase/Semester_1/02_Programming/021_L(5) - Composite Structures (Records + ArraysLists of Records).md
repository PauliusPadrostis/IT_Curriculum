## Context

**Exam link:** “sudėtinės duomenų struktūros” is explicitly listed.

## Sub-topics

- Struct/record modeling (fields, types)

- Array/list of records; record inside collection; collection inside record (basic nesting)

- Selecting field keys for sort/search

- Memory/clarity tradeoffs (not optimization theatre—practical clarity)

## Completion Definition

Student independently:

- Defines a composite data structure (record/struct/class or equivalent) matching a given specification

- Stores parsed file data into a collection of these composite elements

- Correctly accesses and manipulates fields using structured referencing (not parallel arrays)

- Iterates over the collection to:

  - Compute simple aggregates

  - Print formatted structured output

Data model must reflect specification exactly (field names, types, structure).

---

## Minimum Evidence

Student submits:

- Program that:

  - Defines composite structure explicitly

  - Parses file into list/array of composite elements

  - Prints selected fields from first and last record

  - Computes and prints at least one aggregate using structured access

- No use of parallel primitive arrays allowed

- No scaffolded structure template provided

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply structured data modeling

- Analyze specification-to-model alignment

- Implement structured storage and access

Appropriate escalation from raw parsing.

---

## Readiness Gate

✔ All must be true:

- Composite structure matches specification exactly

- All parsed records correctly stored in structure

- No parallel-array implementation

- Field access is correct and type-safe

- Aggregate computation uses structured access

If structure misaligned with spec or parallel arrays used → Gate not met.

---

## Common Failure Modes

- Using multiple primitive arrays instead of composite structure

- Incorrect field ordering or type mismatch

- Storing entire row as string instead of structured fields

- Incorrect structured access syntax

- Forgetting to instantiate structure properly

- Hardcoding field positions without clear mapping
