# I24 (4) — Workflow 2: Records-Based Processing (Read → Select/Filter → Report Output) - Function based

**Type:** 🔗 Integration
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Programming – File I/O, Parsing & Records
**Semester:** Semester 1
**Sequence:** 24 of grade year
**Lessons allocated:** 4
**Status:** 🚧 WIP

---

## Context

Multi-criteria selection on records; output list + summary.

## Completion Definition

Student independently completes a full structured-data workflow by:

- Reading and parsing file data into composite structures

- Applying selection/filter logic based on one or more field conditions

- Producing a correctly formatted report containing only the selected records

- Computing and printing required derived values (if specified)

- Preserving original data integrity (no unintended mutation unless required)

Filtering logic must strictly match specification (correct comparison operators, correct field).

---

## Minimum Evidence

Student submits:

- Full program source

- Input file used

- Output showing:

  - filtered record list

  - correct formatting

  - any required computed summaries

- Verification note stating:

  - total records parsed

  - total records selected

  - explanation of selection condition in one sentence

No scaffolded filtering template.

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply record-field access and filtering logic

- Analyze condition correctness and output validity

Appropriate escalation from basic aggregation.

---

## Readiness Gate

✔ All must be true:

- Records stored in composite structure

- Filtering condition implemented exactly as specified

- Selected record count correct

- Output formatting exact (spacing/order/precision)

- No runtime errors on empty-result case

If filter logic incorrect or record model misused → Gate not met.

---

## Common Failure Modes

- Comparing wrong field

- Using assignment instead of comparison

- String comparison errors (case sensitivity, whitespace)

- Not handling empty-result scenario

- Filtering while mutating collection incorrectly

- Printing all records instead of filtered subset

- Output formatting drift
