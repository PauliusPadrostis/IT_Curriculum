## Context

Focus: composite structure + selection logic.

## Completion Definition

Student independently completes a graded practical task that requires:

- Parsing file data into composite structures (records + collection)

- Applying one or more filtering conditions based on record fields

- Producing a structured report containing:

  - Only selected records

  - Any required derived values (e.g., counts, partial aggregates)

- Exact specification-compliant output formatting

- Safe handling of empty-result cases

This assessment validates correct data modeling + structured selection logic.

---

## Minimum Evidence

Student submits under graded conditions:

- Complete working program

- Input file (provided during assessment)

- Produced output demonstrating:

  - Correct filtered subset

  - Correct record count or derived values (if required)

  - Proper formatting

No scaffolding, no template code, no mid-assessment correction.

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply composite structure + filtering mechanics

- Analyze specification to ensure correct condition logic and output structure

Escalation from aggregate-only tasks to record-level logic.

---

## Readiness Gate

✔ All must be true:

- Composite structure used correctly (no parallel arrays)

- Filtering condition matches specification exactly

- Correct number of records selected

- Empty-result case handled safely (if applicable)

- Output format exactly matches specification

- Program runs without runtime errors

Any structural or filtering error results in rubric-based grade reduction.

---

## Common Failure Modes

- Filtering wrong field

- Incorrect comparison logic (&gt;, &lt;, == misuse)

- Case-sensitivity errors in string comparisons

- Printing all records instead of filtered subset

- Not handling zero-match scenario

- Mutating collection unintentionally during filter

- Output formatting drift

- Hardcoding expected outputs instead of computing dynamically
