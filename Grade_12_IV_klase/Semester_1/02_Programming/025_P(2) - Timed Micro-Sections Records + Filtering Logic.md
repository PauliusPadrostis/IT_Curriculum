## Context

Short tasks with deliberate edge cases; correctness &gt; speed, then speed.

## Completion Definition

Student independently completes short, timed tasks that require:

- Parsing file data into composite structures

- Applying one or more filtering conditions on record fields

- Producing correctly formatted filtered output

- Handling empty-result cases safely

Execution must demonstrate both structural correctness and time-controlled fluency.

---

## Minimum Evidence

Student submits within time limit:

- Working program for each micro-section

- Output showing:

  - Correct filtered results

  - Correct record count after filtering

  - Safe handling of “no records found” case

- Recorded completion time (teacher logged)

No filtering template or scaffold provided.

---

## Cognitive Level (Bloom's)

**Bloom: Apply**

Primary objective is speed + correctness under pressure using structured records.

---

## Readiness Gate

✔ All must be true:

- Composite structure correctly used

- Filtering condition matches specification exactly

- Output formatting exact

- Safe handling of empty-result case

- Completion within defined time threshold

- No runtime errors

If filter incorrect or time exceeded → Gate not met.

---

## Common Failure Modes

- Filtering wrong field

- Incorrect comparison operator

- String comparison issues (case/whitespace)

- Forgetting to handle zero-match case

- Using global variables unnecessarily

- Formatting drift under time pressure

- Mutating collection unintentionally during filter
