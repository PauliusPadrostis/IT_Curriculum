# I10 (2) — Workflow 4: Multi-Question Packet (Mini “Exam Set”)

**Type:** 🔗 Integration
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Data – Reporting & Analysis
**Semester:** Semester 1
**Sequence:** 10 of grade year
**Lessons allocated:** 2
**Status:** 🚧 WIP

---

## Context

**Integration target**

- A sequence of short prompts that force switching techniques (filter → pivot → chart → written inference).

## Completion Definition

Student independently completes a structured multi-part data task under timed conditions that requires:

- Cleaning structural issues

- Applying correct filter/sort logic

- Producing either subtotals or pivot-based report (as specified per question)

- Generating at least one chart where required

- Writing short, numerically grounded conclusions

Student demonstrates procedural independence and logical consistency across multiple question types within one dataset.

---

## Minimum Evidence

Student submits:

- Completed multi-question packet (all required outputs)

- Final dataset state (cleaned and structurally valid)

- All generated reports/charts

- Written responses to each question (short, numeric-based answers)

Conditions:

- No step-by-step prompts

- No structural hints

- Timed execution (teacher recorded)

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze → Evaluate (Exam Conditioning Level)**

- Apply previously mastered mechanics

- Analyze question demands

- Evaluate which method/tool is appropriate per sub-question

This mirrors VBE 2 structural expectations.

---

## Readiness Gate

✔ All must be true:

- All sub-questions answered

- No structural dataset errors remain

- Correct method selected per question

- All numeric outputs verifiable

- No logical contradictions between steps

- Completion within defined time window

If any question is structurally incorrect or incomplete → Gate not met.

---

## Common Failure Modes

- Misreading task requirement (wrong tool selection)

- Time mismanagement (unfinished sections)

- Carrying early structural error into later tasks

- Producing correct report but wrong conclusion

- Skipping numeric references in written responses

- Mixing subtotal and pivot logic inconsistently
