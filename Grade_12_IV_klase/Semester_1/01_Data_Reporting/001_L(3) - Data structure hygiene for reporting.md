# L1 (3) — Data structure hygiene for reporting

**Type:** 📖 Lesson
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Data – Reporting & Analysis
**Semester:** Semester 1
**Sequence:** 1 of grade year
**Lessons allocated:** 3
**Status:** 🚧 WIP

---

## Context

**Why here:** everything later (grouping, pivot, subtotals) breaks without clean structure.

## Sub-topics

- Dataset integrity: headers, types, missing values, duplicates

- Converting “raw export” → analysis-ready table

- Reproducible cleaning log (what changed, why)

## Completion Definition

Student independently transforms a structurally flawed dataset into a reporting-ready table that:

- Contains no merged cells

- Uses one-row header structure

- Has consistent data types per column

- Removes duplicates or empty structural rows

- Uses normalized categorical values (no mixed naming variants)

The output must be technically usable for sorting, filtering, grouping, and pivot operations without structural errors.

---

## Minimum Evidence

Student submits:

- Original flawed dataset

- Corrected cleaned dataset

- A short written justification (3–5 sentences) identifying:

  - At least two structural problems found

  - The correction applied

  - Why the correction was necessary for reporting

No guided prompts. No template hints.

---

## Cognitive Level (Bloom's)

**Bloom: Analyze → Apply**

- Analyze structural flaws

- Apply corrective restructuring

- Justify necessity of correction

Appropriate for Grade 12 pre-report architecture stage.

---

## Readiness Gate

✔ Dataset passes automated structural check:

- Single header row

- No blank rows inside data body

- No merged cells

- Uniform data type per column

- No duplicated records

If any condition fails → Gate not met.

---

## Common Failure Modes

- Confusing formatting cleanup with structural cleanup

- Leaving hidden merged cells

- Fixing visually but not functionally

- Inconsistent categorical labels ("Vilnius", "vilnius", "Vln")

- Removing data instead of restructuring it

- Producing explanation that describes what was done but not why
