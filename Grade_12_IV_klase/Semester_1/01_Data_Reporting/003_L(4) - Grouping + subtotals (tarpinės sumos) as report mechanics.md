**Type:** 📖 Lesson
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Data – Reporting & Analysis
**Semester:** Semester 1
**Sequence:** 3 of grade year

**Status:** 🚧 WIP

## Context

**Grade 12 anchor:** explicit “grupavimas ir tarpinės sumos”.

## Sub-topics

- Grouping rules (what can be grouped, when grouping fails)

- Subtotal logic: counts/sums/averages by group

- Nested grouping and “resetting totals”

- Reading subtotal output as evidence for conclusions

## Completion Definition

Student independently restructures a flat dataset into a grouped report using:

- Correct grouping field selection

- Subtotals (sum / count / average as required)

- Proper group hierarchy (primary and secondary grouping where specified)

- Collapsible/expandable structure (outline logic)

The resulting report must allow:

- Clear comparison between groups

- Accurate subtotal calculations

- Traceability back to raw rows

---

## Minimum Evidence

Student submits:

- Dataset with grouping applied

- Subtotals correctly calculated per group

- A brief verification note stating:

  - Grouping fields used

  - Subtotal function used (sum/count/avg)

  - Total number of groups generated

No template hints. No guided steps.

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply grouping mechanism

- Analyze which aggregation function is structurally appropriate

---

## Readiness Gate

✔ Report meets all structural conditions:

- Correct grouping hierarchy

- Correct subtotal function

- No double-counting

- Grand total matches sum of subtotals

- Groups correctly collapsible

Any mismatch → Gate not met.

---

## Common Failure Modes

- Grouping without sorting first

- Wrong aggregation function (e.g., count instead of sum)

- Misinterpreting category hierarchy

- Breaking dataset structure after grouping

- Not validating subtotal vs grand total consistency

- Manual calculation instead of structural grouping
