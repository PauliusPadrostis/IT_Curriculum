# I3–I4 (2) — Workflow 2: Clean → Pivot report → Chart → Conclusion

**Type:** 🔗 Integration
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Data – Reporting & Analysis
**Semester:** Semester 1
**Status:** 🚧 WIP

---

## Context

**Integration target**

- Pivot report must answer a stated question; chart must match the message.
-

## Completion Definition

Student independently executes an end-to-end pivot-based reporting workflow:

1. Cleans dataset into pivot-ready structure

2. Builds a pivot report matching a given specification (fields + aggregation)

3. Generates an appropriate chart from the pivot output

4. Writes a concise conclusion grounded in the pivot/chart results (with ≥1 numeric reference)

Workflow must be refresh-stable: pivot updates correctly if source data changes.

---

## Minimum Evidence

Student submits:

- Cleaned dataset

- Pivot report (with visible field configuration evidence)

- Chart built from pivot output (not manual selection from raw table)

- 4–6 sentence conclusion including:

  - one key finding

  - ≥1 numeric reference

- One refresh validation (before/after) showing pivot/chart update after a single source change

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze → Evaluate**

- Apply pivot + chart mechanics

- Analyze correct dimensional setup

- Evaluate which claim is supported by the data

---

## Readiness Gate

✔ All must be true:

- Dataset is pivot-ready (single header row, consistent types, no structural breaks)

- Pivot matches specification (field placement + aggregation)

- Chart type fits question (comparison/trend/composition as required)

- Conclusion includes ≥1 correct numeric reference and no unsupported claims

- Refresh validation succeeds (pivot/chart update confirmed)

Any failure → Gate not met.

---

## Common Failure Modes

- Pivot built on dirty structure → wrong counts/aggregations

- Misplacing fields (e.g., metric in Rows instead of Values)

- Wrong aggregation default (count vs sum)

- Chart created from raw data instead of pivot results

- Conclusion overclaims causation or ignores filtering context

- Forgetting refresh → static report mistaken as dynamic
