**Type:** 📖 Lesson
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Data – Reporting & Analysis
**Semester:** Semester 1
**Status:** 🚧 WIP

## Context

**Exam link:** “duomenų pateikimas suvestinėmis lentelėmis” is a named practical type.

## Sub-topics

- Pivot design: dimensions vs measures

- Multi-dimensional pivot interpretation (rows/columns/filters)

- Pivot formatting for a “submit-ready” report

- Common pivot mistakes (double-counting, wrong aggregation)

## Completion Definition

Student independently produces a pivot-based report that:

- Builds a pivot table with correct **Rows / Columns / Values / Filters** placement per specification

- Uses correct aggregation (sum/count/avg) and number formatting

- Applies at least **one pivot-level filter or slicer-equivalent** (tool-dependent) to isolate a reporting slice

- Outputs a clean report layout suitable for submission (title, labels, units, readable structure)

Report must be reproducible: refresh produces correct results if source data changes.

---

## Minimum Evidence

Student submits:

- Source dataset + pivot report sheet

- Pivot configuration evidence (e.g., field list screenshot or exported settings view)

- One refresh validation:

  - Student changes one source value

  - Refreshes pivot

  - Shows that the pivot output updates correctly (before/after captured)

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply pivot construction mechanics

- Analyze correct dimensional design (what belongs where) + aggregation correctness

---

## Readiness Gate

✔ Pivot report meets all:

- Correct field placement (Rows/Columns/Values/Filters) per spec

- Correct aggregation + formatting

- Pivot updates correctly on refresh (demonstrated)

- Layout is readable and report-ready (labels present, no ambiguous units)

If any fails → Gate not met.

---

## Common Failure Modes

- Treating pivot like manual grouping (wrong tool choice)

- Misplacing a field (e.g., metric in Rows instead of Values)

- Wrong aggregation default (count instead of sum)

- Forgetting refresh dependency (static result)

- Producing a pivot but not a usable report layout (missing labels/units/title)

- Filtering the source table instead of pivot-level filtering
