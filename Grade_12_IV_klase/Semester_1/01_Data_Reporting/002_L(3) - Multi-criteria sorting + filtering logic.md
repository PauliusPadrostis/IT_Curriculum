## Context

**Exam link:** multi-criteria sorting/filtering is a core data analysis action type.

## Sub-topics

- Multi-level sorting (primary/secondary keys)

- Multi-criteria filtering (AND/OR logic)

- “Filter-as-question”: turning prompt into selection rules

## Completion Definition

Student independently applies **multi-criteria sorting and filtering** to produce a correct reporting slice from a dataset, where they:

- Select appropriate sort keys (≥2 levels) with correct priority order

- Apply filters using ≥2 conditions (e.g., AND / OR logic) across columns

- Demonstrate correct handling of:

  - numeric vs text sort behavior

  - missing values (blank/NA) without breaking logic

- Produce a final filtered view that matches a given specification exactly

---

## Minimum Evidence

Student submits:

- The dataset (or provided dataset copy)

- A screenshot/export showing:

  - active sort levels (with order)

  - active filters (with conditions visible)

- A short verification note containing:

  - the expected record count

  - the actual record count after filtering

  - one sentence explaining why the primary sort key must precede the secondary key

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply multi-step operations

- Analyze correctness vs specification (priority + logic)

---

## Readiness Gate

✔ Student output matches the specification exactly:

- Correct sort priority order (key1 &gt; key2 …)

- Correct filter logic (AND/OR as required)

- Correct resulting record count (exact match)

- No unintended exclusions due to data-type mistakes

If any mismatch → Gate not met.

---

## Common Failure Modes

- Sorting only one column (ignoring required secondary criteria)

- Reversing priority (secondary key applied as primary)

- Misusing OR instead of AND (or vice versa)

- Filtering by visual guess rather than explicit condition

- Text-sorting numbers (e.g., “100” before “20”)

- Forgetting blanks/NA handling causing silent record loss

- Not verifying record count → undetected errors
