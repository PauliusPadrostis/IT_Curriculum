## Context

Focus: function decomposition + parameter passing + explicit test evidence.

## Completion Definition

Student independently completes a graded, full-length practical task that requires:

- Correct interpretation of specification (multi-part if applicable)

- File input + robust parsing

- Composite structures (records + collection)

- Key-based operations (filter/sort/search and/or mutation as required)

- Modular architecture (clear functional decomposition)

- Robust handling of edge cases

- Exact specification-compliant output formatting

Program must demonstrate full-stack competence under exam-realistic conditions.

---

## Minimum Evidence

Student submits under graded conditions:

- Complete working modular program

- Input dataset provided during assessment

- Output demonstrating:

  - Correct results for all required task parts

  - Correct handling of edge and boundary cases

  - Exact formatting compliance

- No scaffolding

- No mid-task correction

If multi-part task:

- Shared data model must be reused (no duplicate parsing logic)

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze → Evaluate**

- Apply full programming stack

- Analyze task structure and constraints

- Evaluate correctness via invariant preservation and robust handling

This is the peak performance checkpoint of the Programming block.

---

## Readiness Gate

✔ All must be true:

- Program runs without runtime errors

- File parsed correctly

- Data model correct and used consistently

- Required operations implemented correctly

- Modular structure present and compliant

- Edge cases handled correctly

- Output formatting exactly matches specification

- No debug or extraneous output

Failure in architecture, robustness, or formatting results in grade penalty per rubric.

---

## Common Failure Modes

- Misinterpreting multi-part structure

- Re-reading file for second part

- Global state side effects between parts

- Breaking modular separation (logic in `main`)

- Failing on edge cases (empty dataset, duplicate keys, single record)

- Incorrect invariant preservation after mutation

- Minor formatting mismatches under pressure

- Last-minute debugging changes introducing regressions
