# A33 (2) — Practical Assessment 3: Sort/Search + Structured Output

**Type:** 📝 Assessment
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Programming – Sort/Search, Modular & Robustness
**Semester:** Semester 2
**Sequence:** 33 of grade year
**Lessons allocated:** 2
**Status:** 🚧 WIP

---

## Context

Focus: algorithm selection (sorted vs unsorted) and correctness.

## Completion Definition

Student independently completes a graded practical task that requires:

- Parsing file data into composite structures

- Sorting records by a specified key (and secondary key if required)

- Performing search operations (linear or binary as appropriate)

- Producing correctly formatted structured output based on sorted/search results

- Safely handling key-not-found cases

This assessment validates algorithm composition on structured collections.

---

## Minimum Evidence

Student submits under graded conditions:

- Complete working program

- Input dataset provided during assessment

- Output demonstrating:

  - Correct sorted order

  - Correct result for at least one successful search

  - Correct handling of not-found case

  - Exact specification-compliant formatting

No algorithm templates or scaffolding provided.

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply sorting and searching patterns correctly

- Analyze preconditions (e.g., sorted requirement for binary search)

- Ensure invariant preservation and output correctness

This represents mid-to-high complexity graded validation.

---

## Readiness Gate

✔ All must be true:

- Records correctly stored in composite structure

- Sorting matches specification exactly

- Binary search used only when collection sorted (if applicable)

- Search result correct for both found and not-found cases

- Output formatting exactly matches specification

- Program runs without runtime errors

Violation of invariants or incorrect algorithm → grade penalty per rubric.

---

## Common Failure Modes

- Sorting wrong field

- Incorrect comparator direction

- Performing binary search on unsorted data

- Off-by-one indexing errors

- Not handling not-found safely

- Mutating collection unintentionally

- Formatting errors under pressure

- Failing on single-element or empty dataset
