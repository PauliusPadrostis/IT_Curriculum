# P31 (2) — Timed micro-sections: composite structures + key-based operations

**Type:** 🏃 Practice
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Programming – Sort/Search, Modular & Robustness
**Semester:** Semester 2
**Sequence:** 31 of grade year
**Lessons allocated:** 2
**Status:** 🚧 WIP

---

## Context

Records + key selection + sorting/filtering.

## Completion Definition

Student independently completes short, timed programming sections that require:

- Correct use of composite data structures (records + collection of records)

- Performing key-based operations on the collection, such as:

  - search by key

  - sort by key

  - conditional selection by key

- Producing output exactly matching specification

Execution must be both correct and within defined time constraints.

---

## Minimum Evidence

Student submits within time limit:

- Working program for each micro-section

- Output demonstrating:

  - Correct composite structure usage

  - Correct key-based operation (as specified)

  - Correct handling of key-not-found case (if applicable)

- Recorded completion time (teacher logged)

No scaffolded structure or algorithm template provided.

---

## Cognitive Level (Bloom's)

**Bloom: Apply**

Focus is fluent execution of structured data + key-based logic under time pressure.

---

## Readiness Gate

✔ All must be true:

- Composite structure correctly defined and used

- Key-based operation implemented exactly as specified

- Correct result for at least one valid key

- Safe behavior for invalid/not-found key (if specified)

- Output formatting matches specification exactly

- Completion within defined time threshold

- No runtime errors

If structure misused, key logic incorrect, or time exceeded → Gate not met.

---

## Common Failure Modes

- Using parallel arrays instead of composite structure

- Comparing wrong field as key

- Incorrect comparator or condition

- Failing to handle not-found case safely

- Modifying collection unintentionally

- Formatting drift under time pressure

- Crashing on empty or single-element dataset
