## Context

### 1️⃣ Completion Definition (Observable Capability)

Student independently completes a graded **mini-mock exam** that:

- Combines structured **Data practical** and **Programming practical**

- Requires sustained performance across a longer sitting than targeted-prep tasks

- Demonstrates:

  - Stable method selection (Data)

  - Robust file parsing + composite structures (Programming)

  - Correct algorithm composition (if required)

  - Exact formatting discipline in both modalities

- Manages time effectively across sections without abandonment

This assessment validates readiness for **full mock exams**.

## Readiness Gate

✔ All must be true:

- Both Data and Programming sections completed

- No runtime errors (Programming)

- No structural aggregation/model errors (Data)

- Output formatting exact in both sections

- Completed within defined time threshold

- No section abandoned due to time collapse

Failure indicates instability before entering full mock cycle.

## Common Failure Modes

- Time collapse in second half

- Minor formatting errors under fatigue

- Method-selection regression (Data)

- Parsing or invariant failure (Programming)

- Overcorrecting small bugs → losing time

- Skipping verification phase

- Partial completion of one section
