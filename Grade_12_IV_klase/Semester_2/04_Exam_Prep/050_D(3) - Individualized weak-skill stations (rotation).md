# D50 (3) — Individualized weak-skill stations (rotation)

**Type:** 🔁 Diagnostic
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Exam Preparation – VBE Simulation
**Semester:** Semester 2
**Sequence:** 50 of grade year
**Lessons allocated:** 3
**Status:** 🚧 WIP

---

## Context

Three stations per lesson (students rotate):

- Station A: Data workflow correction sprints

- Station B: Programming I/O + output spec sprints

- Station C: Algorithm correctness sprints (sort/search/modify)

## Completion Definition

Student independently rotates through targeted skill stations based on their individual mock-analysis profile, and:

- Identifies their specific weak skill category (e.g., formatting precision, pivot grouping logic, binary search invariants, mutation correctness, time allocation)

- Completes a focused corrective task at each assigned station

- Demonstrates measurable improvement in that specific skill area

- Applies corrected strategy in a short verification task

Each student’s pathway is differentiated but anchored to documented weakness.

---

## Minimum Evidence

For each assigned station, student submits:

- Identified weakness category

- Completed corrective task

- Verification task completed without repeating the weakness

- Brief written note explaining the correction strategy used

No full-class identical task; station assignment must align with prior error data.

---

## Cognitive Level (Bloom's)

**Bloom: Analyze → Apply**

- Analyze personal error pattern

- Apply targeted corrective strategy

This is controlled skill rehabilitation, not new content acquisition.

---

## Readiness Gate

✔ Student correctly identifies own weak-skill category\
✔ Completes corrective station task successfully\
✔ Verification task completed without repeating the targeted error\
✔ Demonstrates observable improvement relative to prior mock

If targeted error reappears → Gate not met.

---

## Common Failure Modes

- Misidentifying weakness (“I just rushed” instead of formatting error)

- Completing station mechanically without internalizing correction

- Passing station task but failing verification

- Rotating too quickly without real skill stabilization

- Treating station as repetition instead of repair
