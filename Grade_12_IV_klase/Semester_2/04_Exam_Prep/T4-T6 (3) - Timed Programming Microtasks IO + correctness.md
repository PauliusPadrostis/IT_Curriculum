## Completion Definition

Student independently completes short, timed programming tasks that require:

- Correct interpretation of a written specification

- Accurate file input and parsing

- Correct implementation of a small, clearly bounded logic requirement

- Exact output formatting compliance

Focus is on eliminating:

- I/O mistakes

- Format errors

- Basic logical inaccuracies

Execution must be both correct and within defined time constraints.

---

## Minimum Evidence

For each timed microtask, student submits:

- Complete working program

- Correctly formatted output

- Logged completion time

Constraints:

- No scaffolding

- No partial code provided

- No teacher correction mid-task

- Output must match specification exactly

Difficulty increases progressively from T4 → T6.

---

## Cognitive Level (Bloom's)

**Bloom: Apply**

Primary objective is fluency and precision under time pressure.

---

## Readiness Gate

✔ All must be true:

- Program compiles/runs without runtime error

- File parsing correct

- Logic correct per specification

- Output formatting exact

- Completed within defined time threshold

If formatting incorrect, logic wrong, or time exceeded → Gate not met.

---

## Common Failure Modes

- Misreading delimiter or input format

- Minor output spacing mistakes

- Off-by-one indexing errors

- Integer division vs float precision issues

- Printing debug statements

- Rushing and skipping edge-case check

- Time mismanagement (spending too long on formatting)
