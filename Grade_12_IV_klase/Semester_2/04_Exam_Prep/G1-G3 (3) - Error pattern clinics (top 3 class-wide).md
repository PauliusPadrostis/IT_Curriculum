## Context

**Typical clusters**

- Data: wrong grouping level / wrong aggregation / misleading chart / unsupported conclusion

- Programming: parsing mistakes / off-by-one / wrong key selection / formatting mismatch / missing edge case\
  (program completeness emphasis).

## Completion Definition

Student participates in structured error clinics where the **top 3 recurring class-wide error patterns** (from mocks) are:

- Diagnosed at root-cause level

- Reconstructed correctly from flawed examples

- Re-implemented correctly under controlled conditions

Student demonstrates ability to:

- Identify the error category

- Explain why it occurs

- Correct it without introducing new regressions

- Apply prevention strategy in a similar task

This sub-topic converts mock performance data into structured correction.

---

## Minimum Evidence

For each of the top 3 error types, student submits:

- Error identification (what category)

- Root cause explanation (why it happens)

- Corrected version of flawed example

- One new similar task solved correctly without repeating the error

No teacher line-by-line debugging provided.

---

## Cognitive Level (Bloom's)

**Bloom: Analyze → Evaluate**

- Analyze systemic error causes

- Evaluate correction validity

- Apply corrective strategy deliberately

---

## Readiness Gate

✔ Student correctly identifies all 3 class-wide error types\
✔ Provides valid root-cause explanation (not just symptom)\
✔ Corrects flawed version successfully\
✔ Does not repeat same error in follow-up task

If error reappears → Gate not met.

---

## Common Failure Modes

- Describing symptom (“wrong answer”) instead of cause

- Blaming time pressure instead of logic flaw

- Fixing formatting but ignoring invariant break

- Applying patch without understanding mechanism

- Repeating same error in new example
