# T7–T9 (3) — Timed Programming Composition: records + sort/search + output spec

**Type:** ⏱ Timed Task
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Exam Preparation – VBE Simulation
**Semester:** Semester 2
**Status:** 🚧 WIP

---

## Context

**Focus**

- Composite structures, key selection

- Sorting + searching choices (sorted vs unsorted)

- Output precision under time

## Completion Definition

Student independently completes timed programming sections that require, end-to-end:

- Parsing file data into **records + collection**

- Performing **sort and/or search** by specified key(s)

- Producing output that matches **exact output specification** (ordering, spacing, conditional cases)

Sections increase in complexity across T7 → T9 by adding:

- secondary keys / tie rules,

- not-found behavior,

- stricter formatting constraints,

- larger datasets (time pressure realism).

---

## Minimum Evidence

For each timed section, student submits:

- Complete working program

- Input file (provided for the section)

- Output text (exact format)

- Logged completion time

Constraints:

- No scaffold code

- No step prompts

- No mid-task correction

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply algorithm composition on structured data

- Analyze specification constraints (keys, ordering, edge cases) under time pressure

---

## Readiness Gate

✔ All must be true:

- Program runs without runtime error

- Records modeled correctly

- Sort/search logic correct per specification

- Not-found / empty cases handled as required

- Output formatting exact

- Completed within defined time threshold

If any condition fails or time exceeded → Gate not met.

---

## Common Failure Modes

- Sorting by wrong field / wrong comparator direction

- Binary search used without sorted precondition

- Tie-handling ignored

- Not-found case crashes or prints wrong message

- Off-by-one indexing errors

- Formatting drift under time pressure

- Over-focusing on speed → skipping verification step
