# T43 (3) — Mixed-section “exam halves” (Data + Programming in one sitting)

**Type:** ⏱ Timed Task
**Grade:** Grade 12 (IV gimnazijos klasė)
**Module:** Exam Preparation – VBE Simulation
**Semester:** Semester 2
**Sequence:** 43 of grade year
**Lessons allocated:** 3
**Status:** 🚧 WIP

---

## Context

**Focus**

- Switching cost training: Data task then Programming task (or reverse)

- Time budgeting strategy + reset routine

- Minimal-error execution

## Completion Definition

Student independently completes a mixed sitting that includes:

- One **Data** section (spreadsheet practical actions + brief conclusion if required)

- One **Programming** section (file I/O + correct logic + exact output formatting)

Student demonstrates:

- Correct context switching without performance collapse

- Time allocation discipline across two different exam modalities

- Maintenance of format precision in both environments

Difficulty escalates across T10 → T12 by increasing dataset/program constraints and tightening time.

---

## Minimum Evidence

For each mixed sitting, student submits:

- Data artifact (sheet/file) with required outputs

- Programming source code + produced output

- Logged completion time for the whole sitting

- No scaffolding, no step prompts, no mid-sitting correction

---

## Cognitive Level (Bloom's)

**Bloom: Apply → Analyze**

- Apply two practical skill stacks under pressure

- Analyze task demands to allocate time and avoid unforced errors

---

## Readiness Gate

✔ All must be true:

- Data section outputs correct and formatted as required

- Programming section compiles/runs and output is exact

- Completed within defined combined time threshold

- No abandonment of one section (both must be completed)

- No cross-task contamination errors (e.g., rushing formatting due to time panic)

If either half fails or time exceeded → Gate not met.

---

## Common Failure Modes

- Over-investing time in first half → second half unfinished

- Switching contexts too slowly (mental reset cost unmanaged)

- Data conclusions unsupported or missing numeric reference

- Programming output formatting errors under fatigue

- Loss of verification habits under mixed pressure

- Panic tool-switching (rewriting instead of finishing)
