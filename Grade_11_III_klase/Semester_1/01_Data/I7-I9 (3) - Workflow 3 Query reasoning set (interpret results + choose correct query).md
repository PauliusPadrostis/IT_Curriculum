# I7–I9 (3) — Workflow 3: Query reasoning set (interpret results + choose correct query)

**Type:** 🔗 Integration
**Grade:** Grade 11 (III gimnazijos klasė)
**Module:** Data – Collection, Joins & Databases
**Semester:** Semester 1
**Status:** 🚧 WIP

---

## Context

Emphasis is **analysis**, not heavy tooling: “what does this query return?” (VBE 1 style).\
\

## Completion Definition

Student can complete a workflow where they:

- Receive a reporting question

- Select the correct query strategy

- Interpret the result set correctly

- Explain what the result implies (and what it does not imply)

This is about mapping:\
**question → query → result → conclusion**

## Minimum Evidence

Student submits a small workflow pack containing:

- 2 reporting questions

- for each:

  - the chosen query (or query plan)

  - the produced/expected result set

  - a 3–4 sentence interpretation including ≥1 numeric reference if applicable

  - one limitation note (e.g., missing data, selection bias, duplicates)

No scaffolding templates.

## Cognitive Level (Bloom's)

**Bloom:** Analyze → Evaluate

## Readiness Gate

✔ All true:

- Query choice matches the question intent

- Interpretation aligns with the result set (no overclaim)

- Includes at least one explicit limitation\
  If not → not ready for P4/A3.
