# I30 (4) — String + array composition tasks

**Type:** 🔗 Integration
**Grade:** Grade 11 (III gimnazijos klasė)
**Module:** Programming – Strings, Arrays & Sort/Search
**Semester:** Semester 2
**Sequence:** 30 of grade year
**Lessons allocated:** 4
**Status:** 🚧 WIP

---

## Context

- Problems that mix string parsing into array counting/aggregation
- Emphasis: transformation pipeline thinking (parse → compute → output)

## Completion Definition

Student can design and implement a task that:
  - Reads structured string input (e.g., list-like format, delimited values, or encoded string)

  - Transforms it into an array representation

  - Traverses the array with correct indexing

  - Applies at least one conditional aggregate or filter

  - Produces structured output matching a specification

  Student demonstrates understanding that:
  - Strings and arrays share iteration principles

  - Index discipline governs both

  - Loop control logic transfers across representations

  ---

## Minimum Evidence

Student independently completes a composition task that:
  - Parses string data into array form OR processes an array and outputs formatted string report

  - Uses at least one loop-based validation check

  - Applies at least one conditional computation

  - Produces formatted output with no hardcoded values

  - Handles at least one malformed input case

  No guided scaffolding.\
  No partially written loops.

  ---

## Cognitive Level (Bloom's)

**Bloom:** Apply → Analyze
  - Apply traversal logic correctly

  - Analyze data structure transformation (string → array or array → formatted string)

  ---

## Readiness Gate

✔ All must be true:
  - Loop bounds correct and stable

  - No index misuse (e.g., accessing beyond length)

  - String-to-array transformation logically sound

  - Conditional aggregate produces correct result for edge cases

  - Output matches specification exactly

  If any condition fails → not ready for timed micro-sections (P1–P2).
