# A1 (1) — Practical assessment: file-driven algorithm task

**Type:** 📝 Assessment
**Grade:** Grade 11 (III gimnazijos klasė)
**Module:** Programming – Strings, Arrays & Sort/Search
**Semester:** Semester 2
**Status:** 🚧 WIP

---

## Context

- Includes: read → represent → compute (arrays/strings) → output
- Clear criteria: correctness, formatting, edge-case handling
-

## Completion Definition

Student independently implements a complete program that:

- Reads structured data from a `.txt` or `.csv` file

- Parses and stores data correctly (array-based representation)

- Applies at least:

  - one aggregate computation

  - one conditional filter or selection

- Produces formatted output matching a provided specification exactly

- Handles at least one input irregularity (e.g., empty line, malformed value, unexpected token)

No guided scaffolds.\
No partially written loops.\
No template solution bodies.

Program must demonstrate:

- Correct traversal logic

- Correct indexing discipline

- Clear separation of reading, processing, and output stages

---

## Minimum Evidence

Submission must include:

- Source code

- Output file or console output

- The input file used

- Short comment at top describing:

  - data structure used

  - how invalid data is handled

Task must require:

- ≥10 data records

- At least one non-trivial conditional (not just `if x > 0`)

---

## Cognitive Level (Bloom's)

**Bloom:** Apply → Analyze

- Apply traversal and parsing correctly

- Analyze specification to design correct logic

This is the first true “independent algorithm build” checkpoint of Grade 11.

---

## Readiness Gate

✔ All must be true:

- File read correctly (no hardcoded data)

- No index errors

- Aggregate values correct for provided dataset

- Conditional logic correctly implemented

- Output format matches specification exactly (spacing, ordering, labels)

- Program runs without runtime error on provided dataset

If any fail → not ready for Mini-block 2 (Sorting/Search Systems).
