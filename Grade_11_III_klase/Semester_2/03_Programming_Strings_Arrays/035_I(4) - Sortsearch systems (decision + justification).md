# I35 (4) — Sort/search systems (decision + justification)

**Type:** 🔗 Integration
**Grade:** Grade 11 (III gimnazijos klasė)
**Module:** Programming – Strings, Arrays & Sort/Search
**Semester:** Semester 2
**Sequence:** 35 of grade year
**Lessons allocated:** 4
**Status:** 🚧 WIP

---

## Context

- Students must choose: linear vs sorted search; justify choice
- Correctness-focused tasks; code-reading comparisons (VBE 1 alignment)\
  \

## Completion Definition

Student can, for a small dataset (≤ 20 elements):
  - Decide whether sorting is necessary before searching

  - Select appropriate search strategy (linear vs binary)

  - Justify the decision in concise written form

  - Combine sort + search in a coherent workflow

  - Explain consequences of skipping sorting when required

  Student demonstrates system-level thinking:

  > “What must be true before this algorithm works?”

  ---

## Minimum Evidence

Student independently completes integrated tasks that include:
  - Given unsorted array → determine search strategy

  - Given sorted array → justify binary search usage

  - Perform:

    - sort (if needed)

    - then search

  - Provide short-answer justification (1–2 sentences)

  - Identify logical flaw in a system that attempts binary search on unsorted data

  No step-by-step hints.\
  No scaffolding of logic order.

  ---

## Cognitive Level (Bloom's)

**Bloom:** Analyze
  - Analyze preconditions for algorithm validity

  - Evaluate correct system sequence

  ---

## Readiness Gate

✔ All must be true:
  - Correctly distinguishes sorted vs unsorted requirements

  - Applies sort before binary search when necessary

  - Justification logically correct and concise

  - No index or boundary error in combined workflow

  - Identifies at least one invalid system setup correctly

  If any fail → not ready for Insert/Delete operations block.
