**Type:** 🏃 Practice

## Context

- Output prediction, dry-run tables, identify error type
- Short-answer format conditioning\
  \

## Completion Definition

Student can, under time pressure:
  - Read a short program fragment (strings/arrays/loops)

  - Predict output exactly

  - Produce a correct dry-run table (index, key variables, array states where relevant)

  - Identify the *type* of error or risk in a given snippet (e.g., off-by-one, wrong loop bounds, wrong accumulator init, wrong condition placement), even if not fixing it

  - Answer in **short-answer format** (concise, exam-like)

  ---

## Minimum Evidence

Student completes a timed packet containing:
  - ≥2 tracing questions (predict output)

  - ≥1 dry-run table question

  - ≥1 “identify error type” item

  Evidence must include:
  - final output string/lines exactly as expected

  - dry-run table with consistent step indexing

  - error type label + 1-sentence justification

  No collaborative solving. No step-by-step hints.

  ---

## Cognitive Level (Bloom's)

**Bloom:** Analyze
  - Analyze control flow and state changes across iterations

  - Diagnose error category from symptoms/structure

  ---

## Readiness Gate

✔ All must be true:
  - Output prediction exact (format + values)

  - Dry-run table correctly tracks state across all iterations (no skipped steps)

  - Correctly identifies at least one real error type/risk in a flawed snippet

  - Responses are concise and exam-format (no paragraph explanations)

  If any condition fails → not ready for A1 (practical assessment) in this mini-block.
