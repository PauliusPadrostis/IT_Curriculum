# L33 (5) — Sorting + searching (sorted vs unsorted reasoning)

**Type:** 📖 Lesson
**Grade:** Grade 11 (III gimnazijos klasė)
**Module:** Programming – Strings, Arrays & Sort/Search
**Semester:** Semester 2
**Sequence:** 33 of grade year
**Lessons allocated:** 5
**Status:** 🚧 WIP

---

## Context

**Grade 11 anchor:** “rikiavimas ir paieška (rikiuotuose/nerikiuotuose)”.

## Sub-topics

- Linear search vs search in sorted sequence (when/why)

- Sorting by key; stable intuition (practical, not theoretical)

- Correctness reasoning: invariants (“after sort…”, “during search…”)

- Reading and predicting code for sort/search (VBE 1 style)

## Completion Definition

Student can:

- Explain the difference between searching in sorted vs unsorted arrays

- Implement:

  - linear search (unsorted case)

  - one basic comparison-based sort (e.g., selection or bubble)

- Explain why binary search requires sorted data

- Trace and predict search results across different data states

- Justify algorithm choice for a given scenario

Student demonstrates both:

- procedural control over implementation

- reasoning about algorithm suitability

---

## Minimum Evidence

Student independently completes tasks that include:

- Writing linear search from scratch

- Writing one simple sort algorithm from scratch

- Given a dataset, selecting the appropriate search method and justifying in 1–2 sentences

- Predicting number of comparisons in small-scale examples

- Tracing one pass of a sort algorithm correctly

No pre-filled loop bodies.\
No built-in `.sort()` for core implementation tasks (can compare later).

---

## Cognitive Level (Bloom's)

**Bloom:** Apply → Analyze

- Apply algorithm implementation correctly

- Analyze sortedness constraints and efficiency reasoning

---

## Readiness Gate

✔ All must be true:

- Linear search correctly returns index or not-found

- Sorting algorithm produces fully sorted result

- Student can explain (verbally or written) why binary search fails on unsorted data

- No infinite loop or boundary errors

- Can justify algorithm choice for a given task

If any fail → not ready for Insert/Delete systems (L15–L17).
