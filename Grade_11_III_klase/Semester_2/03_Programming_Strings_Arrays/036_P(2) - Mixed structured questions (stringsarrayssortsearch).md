## Context

“Which fragment fits?”, “what changes if…?”, “choose correct sequence”

## Completion Definition

Student can, under time pressure:

- Read a short code fragment (array/string/sort/search logic)

- Predict output or state after execution

- Complete missing lines of code correctly (loop condition, accumulator, swap logic, etc.)

- Identify logical flaw in partial implementation

- Provide short-answer justification for algorithm choice

Student demonstrates integrated understanding across:

- traversal

- sorting

- searching

- system sequencing

---

## Minimum Evidence

Timed packet includes:

- ≥1 output prediction task

- ≥1 “complete the missing line” task

- ≥1 “identify flaw” task

- ≥1 short-answer justification (algorithm choice or precondition reasoning)

Responses must be concise.\
No paragraph explanations.\
No debugging essays.

- Analyze algorithm correctness

- Evaluate system validity

## Readiness Gate

✔ All must be true:

- Output predictions exact

- Missing-line completions logically correct

- Identifies at least one structural flaw correctly

- Justification concise and logically sound

- No boundary/index misunderstanding

If any fail → not ready for A2 (Analytical Assessment).
