# L1–L4 (4) — Strings as data structures (not “text typing”)

**Type:** 📖 Lesson
**Grade:** Grade 11 (III gimnazijos klasė)
**Module:** Programming – Strings, Arrays & Sort/Search
**Semester:** Semester 2
**Status:** 🚧 WIP

---

## Context

**Why here:** strings become the simplest “structured data” before arrays.\
\

## Sub-topics

- Indexing, slicing, traversal patterns

- Frequency/counting patterns on characters

- Comparisons and normalization (case, spaces)

- Typical pitfalls: off-by-one, empty string, non-letter symbols

## Completion Definition

Student manipulates string data programmatically to:

- Extract substrings

- Locate character positions

- Transform case or format

- Validate structured patterns (e.g., ID formats)

- Combine strings with numeric data correctly

No manual “text typing tasks.” All manipulation must occur via algorithmic operations.

---

## Minimum Evidence

Student independently writes a short program that:

- Reads at least one string input

- Performs at least two distinct string operations

- Produces formatted output that combines string + numeric value

- Handles at least one invalid-format case correctly

No guided scaffolds. No pre-written function bodies.

---

## Cognitive Level (Bloom's)

**Bloom:** Apply → Analyze

- Apply string operations correctly

- Analyze input format to decide correct manipulation strategy

---

## Readiness Gate

✔ All must be true:

- Uses built-in string methods correctly (not manual looping unless justified)

- Output format matches specification exactly

- Invalid input handled explicitly

- No hardcoded result logic

If any condition fails → Not ready for array traversal block.
