## Sub-topics

- Standard scan patterns (count / sum / min/max / argmin/argmax)

- Conditional accumulation (“count if…”)

- Multi-pass logic (derive then use)

- Trace tables (manual execution as exam skill)

## Completion Definition

Student can:

- Traverse a 1D array using controlled indexing

- Compute aggregates (sum, count, min, max, conditional count)

- Apply filtering conditions inside traversal

- Update elements safely

- Explain loop boundary correctness

- Construct a simple 2D array and access elements using nested loops

Traversal must be disciplined:

- No off-by-one errors

- No unnecessary nested loops

- No redundant passes unless justified

---

## Minimum Evidence

Student independently writes a program that:

- Processes an array of ≥ 10 elements

- Computes at least:

  - one aggregate

  - one conditional aggregate

- Modifies at least one element based on condition

- Prints structured output

- Includes a short explanation (2–3 sentences) of why loop bounds are correct

For 2D:

- Access at least one row and one column element correctly.

---

## Cognitive Level (Bloom's)

**Bloom:** Apply → Analyze

- Apply traversal correctly

- Analyze condition placement and boundary correctness

---

## Readiness Gate

✔ All must be true:

- Loop bounds correct (no index error)

- Aggregate logic correct for edge cases (all positive / all negative / empty condition)

- No duplicated traversal unless required

- 2D indexing consistent (row/column not swapped)

If any fail → not ready for sorting/search systems.
