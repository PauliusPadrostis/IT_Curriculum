# Project: informatika-curriculum

## Obsidian Vault Context

**Vault**: `obsidian-brain`

### Session Start — Read These Notes

At the beginning of every session, read the following notes via the Obsidian MCP before doing any work:

1. `00-system/active-projects.md` — current project priorities and status links
2. `10-projects/it-curriculum/status.md` — latest progress on this curriculum project
3. `10-projects/it-curriculum/decisions.md` — locked decisions log (append-only, never edit or delete existing entries)

Use the information from these notes to understand current priorities, blockers, and constraints before acting.

### Respecting decisions.md

`decisions.md` is an **append-only log**. Existing entries are locked:
- **Never** edit or delete a previous decision entry.
- **Only** append new dated entries at the bottom when a new decision is made during the session.

### Session End — Update Vault

Before the session ends (when the user signals completion or says goodbye):

1. **Append to `status.md`**: Add a dated summary (`## YYYY-MM-DD`) describing what was accomplished, what changed, and any new blockers.
2. **Append to `decisions.md`**: If any new decisions were made during the session, append them as dated entries following the existing format (`## YYYY-MM-DD — Decision Title` with Decision, Context, and Rationale fields).

Use the Obsidian MCP `edit-note` tool to append content. Do not overwrite existing content.
