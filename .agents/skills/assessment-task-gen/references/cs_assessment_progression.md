# CS Assessment Progression

Grade-appropriate assessment defaults. The A lesson README always overrides this document when it specifies format, question types, or constraints.

---

## Assessment Defaults by Grade

| Grade | Domains | Primary Question Types | Code Task Ceiling | Duration | Notes |
|-------|---------|----------------------|-------------------|----------|-------|
| 9 | Digital literacy, safety, graphics, file management | MCQ, short answer, practical tool tasks (Inkscape, Word) | No programming | 30-35 min | Testing tool usage and concepts. Maximum scaffolding in task instructions. |
| 10 | Intro C++, data/Excel, HTML basics | Code tracing, output prediction, code completion, short answer, spreadsheet tasks | Completion (fill in marked sections) | 35-40 min | Code writing only as small extensions of given code. Spreadsheet tasks use real data. |
| 11 | Intermediate C++, data workflows, theory modules | Code writing (small programs), debugging, file-driven tasks, scenario analysis, data workflows | Write small independent programs | 40-45 min | VBE-adjacent formats for programming. Multi-step data processing tasks. |
| 12 | Advanced C++, data reporting, capstone | Modular programs, multi-part tasks, edge case handling, robustness testing | Write modular programs with functions | 45 min | Full VBE format alignment. Integration tasks across domains. |

---

## Code Task Progression Detail

| Grade | Reading | Tracing | Completion | Writing |
|-------|---------|---------|------------|---------|
| 9 | N/A | N/A | N/A | N/A |
| 10 | Yes | Yes (simple: linear + one control structure) | Yes (primary format) | Only as small extensions |
| 11 | Yes | Yes (loops, arrays, strings) | Yes | Yes (small independent programs) |
| 12 | Yes | Yes (file I/O, structs, functions) | Yes | Yes (modular programs with functions) |

**Rule:** Students who cannot trace at >50% accuracy are not ready to write code (Lister et al.).

## Assessment Format by Domain

| Domain | Recommended Format | Platform | Typical Structure |
|--------|-------------------|----------|-------------------|
| Theory (safety, networks, ethics, digital literacy) | Testmoz test | Testmoz | MCQ + short answer + scenario analysis |
| Programming (C++, algorithms, data structures) | Practical coding task | Code::Blocks | Code completion/writing + rubric |
| Tool-based (Excel, Inkscape, Canva, Word) | Practical product task | Native tool | Create/modify artifact + per-criterion rubric |
| Data (spreadsheets, data analysis, visualization) | Practical spreadsheet task | Excel | Structured data manipulation + rubric |
| Mixed (theory + tool or theory + programming) | Combined format | Testmoz + tool | Testmoz test (theory) + practical task (application) |

**Rule:** When a module covers multiple domains, prefer the mixed format. Split points between theory and practical portions.

---

## VBE Alignment Notes (Grades 11-12)

For grades 11-12 programming assessments:
- Match VBE task structure: problem description, input format, output format, sample I/O, constraints
- Include edge case handling requirements
- Specify time/memory limits only if relevant to the curriculum
- Use realistic input sizes (10-100 records for Grade 11, 100-1000 for Grade 12)
